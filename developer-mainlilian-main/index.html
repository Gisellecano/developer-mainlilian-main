    • Se crea la paleta de colores de manera global. Se crea un pseudo-selector, por ejemplo, root es un elemento que no existe como tal, pero que sirve para almacenar variables y se le conoce como custom properties. Una ventaja es que, si a lo largo del proyecto existe un cambio en algún color de la paleta, no se tendría que cambiar dicho color en todas las partes del código, sino únicamente en la definición de variables del custom properties.   • 21. Empezando con Responsive Web Design (diseño web adaptable o responsivo).


    • Espaciado en Enlaces

    Un enfoque le pedía al usuario poner una url diferente depediendo del dispositivo con el que se esté usando la aplicación web. Facebook lo utiliza actualmente. Aunque realmente existía el problema de que, en lugar de mantener un sitio web, mantenías 3 o 4.

    https://facebook.com/
    https://m.facebook.com/








    Google puede penalizar a los sitios web que no son responsivos. Desde 2015, Google ha hecho cambios en su algoritmo para optimizar la experiencia de los usuarios móviles. Las páginas web que no están optimizadas para dispositivos móviles pueden ser penalizadas en los resultados de búsqueda.

    Esto significa que si el sitio web no está adaptado para visualizarse correctamente en dispositivos móviles, podría aparecer más abajo en los resultados de búsqueda de Google cuando los usuarios busquen desde sus teléfonos o tablets.

    Aquí hay algunas recomendaciones para evitar ser penalizado:

  • 1. Google prefiere sitios web cuya URL es única, independientemente del dispositivo desde el que se visite (ordenadores, tablets, móviles).
  • 2. Evita el contenido duplicado. Google recomienda utilizar el mismo código HTML para todos los dispositivos, adaptando la experiencia de usabilidad vía técnicas de web responsive y siempre bajo una misma URL.
  • 3. Las redirecciones son lentas. La penalización de rendimiento para ejecutar un redireccionamiento puede ser grave en los smartphones.

                  .campo:nth-child(3),
                  .campo:nth-child(4){
                      grid-column: 1/3; /* Para posicionar el elemento, "que vaya de la columna 1 a la 3*/
                  }
              }

  • 39. Agregar la propiedad scroll-snap
    • El Scroll Snap es una propiedad de CSS que permite controlar el desplazamiento en una página web. Al hacer scroll, ya sea con el ratón desde un ordenador o portátil, o desde la pantalla de un smartphone, este desplazamiento se detiene en un punto en concreto de la web, normalmente el comienzo de una nueva sección. Esto es muy útil en sitios web que tienen una disposición en diferentes bloques horizontales, y se quiere que los usuarios salten de uno a otro de la forma más directa posible. Un buen ejemplo de una página que utiliza Scroll Snap es la página de Tesla. Al navegar por ella, se puede observar cómo se salta directamente de una imagen a otra de forma directa. Es importante mencionar que la compatibilidad de Scroll Snap con los navegadores web es bastante amplia, asegurando un buen funcionamiento en casi el 94% de los usuarios. Sin embargo, hay que tener en cuenta que puede haber problemas con Internet Explorer y los navegadores para móviles de Opera. Otro ejemplo además de la página de Tesla, es la página de los Airpods Pro de apple que utiliza Scroll Snap para crear una animación que cambia a medida que el usuario se desplaza por la página. Para manejar en qué parte se estaría aplicando el scroll, se añadiría una clase para el h2 de “Mis servicios” y “Contacto”: 
              
              <h2 class="servicios-heading">Mis servicios</h2>
          
      
            
            <h2 class="contacto-heading">Contacto</h2>
    En la etiqueta global se añade la propiedad del scroll snap y en la parte inferior se definen las propiedades más específicas correspondientes a los títulos y encabezados: 
              
              /* Globales */
              html {
                font-size: 62.5%;
                box-sizing: border-box; /* Hack para el manejo de box model */
                scroll-snap-type: y mandatory; /* La letra "y" hace referencia al eje y que quiere decir que se enfoca en el scroll vertical. Mandatory define que tan estricto se requiere el scroll. Otra alternativa menos estricta es proximity */
              }
      
              /* Scroll snap */
              .titulo,
              .servicios-heading,
              .contacto-heading {
                scroll-snap-align: start; /* Indica el punto en el contenedor de desplazamiento al que debe ajustarse el contenido*/
                scroll-snap-stop: always; /* Controla si el desplazamiento debe detenerse siempre en los puntos de ajuste */
              }

    • Existen muchas páginas para subir proyectos web como Hostinger, HostGator y demás; sin embargo, la mayoría son de paga y no ofrecen muchas opciones a un plan gratuito. Otra opción más simple podría ser Github Pages, para fines educativos funciona bien. La alternativa más recomendada hoy en día es netlify ya que ofrece las siguientes ventajas:
    • 1. Es totalmente gratis probarla
    • 2. Es muy fácil subir los proyectos así como su actualización.
    • 3. No tiene publicidad
    • 4. Netlify puede vincular múltiples nombres de dominio, mientras que GitHub Pages solo puede vincular un nombre de dominio.
    • 5. Con Netlify, el repositorio original puede ser privado, mientras que GitHub Pages requiere que el repositorio original sea público.
    • 6. Primero se crea una cuenta en Netlify utilizando la cuenta de GitHub. Para ello se requerirá autorizar ciertos permisos. Si por alguna razón no se puede crear la cuenta vinculándolo con GitHub, se tendría que crear por correo.




    • 7. Se accede a la opción “Sites” que está en el panel de la izquierda.


    • 8. Se arrastra y suelta la carpeta que contiene el proyecto en el espacio correspondiente. No se recomienda subir los archivos por separado.
    • 9. Al esperar unos cuantos segundos, ya deberá funcionar y se accede a la opción “Site overview” que está en el panel de la izquierda.
    • 10. Si se necesita hacer deploy de algún cambio en el proyecto, se requiere volver a cargar la carpeta completa en la opción “Deploy” que está en el panel de la izquierda.
    • 11. Para cambiar el nombre del dominio para usar uno personalizado, se accede a la opción “Site configuration” que está en el panel de la izquierda y ahí se presiona el botón “Change site name”.
